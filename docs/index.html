Visão Geral do Sistema

O que é o CBAV?

O CBAV (Congregação Batista Avenida) é um sistema CRM ministerial completo desenvolvido pela Vertex Solutions. Ele oferece uma solução integrada para gestão de igrejas, incluindo membros, eventos, finanças, EBD digital e muito mais.

Características Principais:

  • Gestão completa de membros
  • Sistema financeiro integrado
  • EBD digital com certificados
  • Sistema de eventos
  • Conselho ministerial
  • Devocionais e Bíblia digital

Tecnologias:

  • Laravel 10.x
  • PHP 8.2+
  • MySQL 8.0
  • JavaScript ES6+
  • Tailwind CSS
  • Git Version Control

Arquitetura do Sistema

Estrutura MVC

O sistema segue o padrão MVC (Model-View-Controller) do Laravel, proporcionando organização e manutenibilidade do código.

Models

Representam as entidades do banco de dados e contêm a lógica de negócio.

  • • User (Usuários)
  • • Membro (Membros)
  • • Transacao (Transações)
  • • Evento (Eventos)
  • • EbdAluno (Alunos EBD)

Views

Interface do usuário criada com Blade templates e Tailwind CSS.

  • • Admin Views
  • • Member Views
  • • Public Views
  • • Email Templates
  • • PDF Templates

Controllers

Controlam o fluxo de dados entre Models e Views.

  • • Admin Controllers
  • • Member Controllers
  • • Public Controllers
  • • API Controllers
  • • Gateway Controllers

Estrutura de Diretórios

CBAV/
├── app/
│   ├── Http/Controllers/
│   │   ├── Admin/           # Controllers administrativos
│   │   ├── Member/          # Controllers de membros
│   │   └── Public/          # Controllers públicos
│   ├── Models/              # Models do sistema
│   ├── Services/            # Serviços de negócio
│   └── Helpers/             # Helpers e utilitários
├── resources/views/
│   ├── admin/               # Views administrativas
│   ├── member/              # Views de membros
│   ├── public/              # Views públicas
│   └── layouts/             # Layouts base
├── routes/
│   └── web.php              # Rotas do sistema
├── database/
│   ├── migrations/          # Migrações do banco
│   └── seeders/             # Seeders de dados
└── docs/                    # Documentação
    └── index.html           # Esta página
